All'esecuzione di questo programma, il call stack al suo apice sar rappresentato nel modo seguente:

In realt questa rappresentazione semplificata, in quanto ogni invocazione di funzione, oltre ai parametri attuali, memorizza nello stack anche il return address della funzione (assieme ad altre informazioni non direttamente accessibili).

Attraverso il return address, il programma pu tornare al punto esatto in cui si era interrotto e proseguire l'esecuzione dell'invocante. In altri termini, data una invocazione di funzione, il suo return address l'indirizzo, nella funzione invocante, dell'istruzione immediatamente successiva all' invocazione.

Nel esempio di sopra, 0x1202 sar l'indirizzo dell'istruzione immediatamente successiva all'invocazione get_media(a,b); nella funzione main(...), mentre 0x1600 sar l'indirizzo dell'istruzione immediatamente successiva all'invocazione get_somma(a, b); nella funzione get_media(a,b);

Ovviamente, termine di ogni funzione l'intero stack frame verr eliminato, compreso l'indirizzo di ritorno.

$\Rightarrow$ ogni invocazione di funzione, anche se non porta parametri, costa in termini di spazio sul call stack.

Funzione ricorsiva¶

Funzione ricorsiva (programmazione): strumento messo a disposizione da tanti (ma non tutti) i linguaggi di programmazione (e.g., FORTRAN 77 non supporta la ricorsione) che permette ad una funzione di "invocare se stessa"

Esempio:

In [ ]:
#include <stdio.h>

void f()
{
    printf("ciao\n");
    f();
}

int main()
{
    f();
    return 0;
}

Nel caso precedente, la funzione f() invoca se stessa (teoricamente) all'infinito. Ad ogni invocazione, la funzione invocante:

  • salva nello stack di esecuzione il punto in cui si trova (cos come qualsiasi altra funzione)
  • invoca s stessa (quindi ricomincia dall'inizio)
  • una volta terminata l'esecuzione, ritorna al punto in cui l'invocante si era interrotto

$\Rightarrow$ necessario un criterio di stop!

Da notare che, ad ogni invocazione, la funzione invocante deve inviare tutte le variabili di cui necessita l'invocata, ossia:

  • dati che specificano su quale/i elemento/i lavorare (in questo caso, variabile i)
  • dati necessari per effettuare i calcoli (in questo caso, max)

Problema: Scrivere una funzione ricorsiva per il calcolo del fattoriale di un numero.

Si ricorda che il fattoriale di un numero $n$

  • $1$ se $n=0$,
  • altrimenti dato da $n!=\prod\limits_{i=1}^n i = 1 \cdot 2 \cdot 3 \cdot\dots \cdot n$

Una possibile soluzione iterativa al calcolo del fattoriale in un numero $n$ la seguente:

int fattoriale(int n)
{
    int i    = 1;
    int fatt = 1;
    while(i <= n)
    {
        fatt = fatt * i;
        i++;
    }
    return fatt;
}

una possibile conversione della funzione precedente in funzione ricorsiva la seguente:

Funzione Ricorsiva Vs. Soluzione Ricorsiva¶

Funzione ricorsiva (programmazione): strumento messo a disposizione da tanti (ma non tutti) i linguaggi di programmazione (e.g., FORTRAN 77 non supporta la ricorsione) che permette ad una funzione di "invocare se stessa"

E' importante distinguere tra una funzione ricorsiva ed una soluzione ricorsiva:

Soluzione ricorsiva: soluzione ad un problema che sfrutta una possibile suddivisione di quest'ultimo in sottoproblemi pi semplici da risolvere (generalmente di dimensione ridotta)

NB: L'utilizzo di una funzione ricorsiva non implica che si stia fornendo una soluzione ricorsiva

COROLLARIO: Se vi si chiede una soluzione ricorsiva ad un problema e voi presentate un funzione ricorsiva che non implementa una soluzione ricorsiva, non state presentando quanto richiesto.

Esempio: voglio una soluzione ricorsiva al calcolo del fattoriale di un numero $n$.

Per costruire una soluzione ricorsiva ad un problema una strategia pu essere quella di porsi le seguenti domande:

  1. esistono dei casi in cui il mio problema facilmente risolvibile (ossia che posso considerare banale)?
  2. posso sfruttare le soluzioni ai casi banali individuati per costruire soluzioni ad un caso un po' pi complesso?
  3. posso sfruttare la mia nuova soluzione per costruire una soluzione ad un caso ancora pi complesso?
  4. posso generalizzare?

Nel caso specifico, esistono dei numeri per cui il calcolo del fattoriale particolarmente semplice da calcolare?

La risposta s: sappiamo infatti che per $n=1$ (ed anche per $n=0$) il calcolo del fattoriale banale, ossia che $0!=1$ ed $1!=1$.

In realt, sapendo che $0!=1$, posso anche scrivere $1!=1 \cdot 0!$.

  1. posso sfruttare le soluzioni ai casi banali individuati per costruire soluzioni ad un problema un po' pi complesso?

vediamo: se $n=2$ il fattoriale sar $2! = 2 \cdot 1 = 2$ $\Rightarrow 2!=2 \cdot 1 = 2 \cdot (2-1)!$

Quindi ho utilizzato la soluzione $1!$ per risolvere il problema $2!$

  1. posso sfruttare la mia nuova soluzione per costruire una soluzione ad un caso ancora pi complesso?

vediamo: se $n=3$ il fattoriale sar $3! = 3 \cdot 2 \cdot 1 = 3 \cdot 2!=3 \cdot(3-1)!$

Quindi ho utilizzato la soluzione $2!$ per risolvere il problema $3!$

se $n=4$ il fattoriale sar $4! = 4 \cdot 3 \cdot 2 \cdot 1 = 4 \cdot 3!=4 \cdot(4-1)!$.

  1. posso generalizzare?

In generale, $$\prod\limits_{i}^{n}i = n \cdot \prod\limits_{i}^{n-1}i$$ quindi

$$\forall n \geq 1,\ n!=n \cdot (n-1)!$$

Posso quindi riscrivere il fattoriale di un numero come:

$$ n!=\begin{cases} 1 & \text{se } n = 0 \\ n \cdot (n-1)! & \text{se } n > 0 \end{cases} $$

Tale soluzione implementabile in maniera (quasi) immediata attraverso una funzione ricorsiva:

int fattoriale(int n)
{
    if(n == 0)
        return 1;
    return n* fattoriale(n-1);
}

Problema: Dato un insieme di numeri $S$, determinarne il massimo. Proporre una soluzione ricorsiva

soluzione iterativa: scorro attraverso un ciclo gli elementi del mio insieme,confrontando ogni elemento con il massimo locale trovato fino a quel momento.

implementazione iterativa:

int A[] = {7,6,8,3}; //NB: in generale, si ricorda che un vettore non  un insieme
int n   = 4;
int m   = A[0];
for(int i=1; i < n; i=i+1)
{
    if(A[i] > m)
        m = A[i];
}
  1. esistono dei casi in cui il mio problema facilmente risolvibile?

Il problema cercare il massimo in un insieme di numeri, ossia si vuole trovare una funzione $max\_set(S)$ che restituisca il pi grande da un insieme di numeri $S$ dato in input.

Intuitivamente, pi l'insieme $S$ piccolo, pi il problema semplice.

In particolare, il problema risulta particolarmente semplice se $|S|=1$ oppure $|S|=2$.

  • se $S$ ha un solo elemento, i.e. $S=\{s\}$, allora il pi grande banalmente $s$. Posso dire che: $$\text{ se } S=\{s\},\ max\_set(S)=s $$
  • se $S$ ha due elementi, i.e. $S=\{a,b\}$, allora il pi grande elemento nell'insieme il massimo tra i due elementi, che posso calcolare facilmente.
$$ \text{ se } S=\{a,b\}, max\_set(S)=\max(a,b)$$

con $\max(\cdot,\cdot)$ funzione di appoggio definita come $\max(a,b)=\begin{cases} a & \text{se } a \geq b\\ b & \text{altrimenti}\\ \end{cases}$

  1. posso sfruttare le soluzioni ai casi banali individuati per costruire soluzioni ad un caso un po' pi complesso?

Il caso immediatamente pi complesso quello avente $|S|=3$. Dato che so risolvere il problema per $|S|=1$ ed $|S|=2$, posso sfruttare queste soluzioni per risolvere il caso $|S|=3$?

Il massimo tra 3 numeri $S$ lo posso vedere come il massimo tra due numeri qualsiasi in $S$ ed il numero rimanente, ossia:

$$ \text{ se } S=\{a,b,c\},\ max\_set(S)=\max(c,\max(a,b))$$

Dato che so che $\max(a,b)= max\_set(\{a,b\})$,

possiamo quindi riscrivere la soluzione al problema come:

$$ \text{ se } S=\{a,b,c\}, max\_set(S)=\max(c,max\_set(\{a,b\}))$$
  1. posso sfruttare la mia nuova soluzione per costruire una soluzione ad un caso ancora pi complesso?

se $|S|=4$, posso ripetere il ragionamento di prima e sfruttare la soluzione avuta con $|S|=3$, quindi

$$ \text{ se } S=\{a,b,c,d\}, max\_set(S)=\max(d,max\_set(\{a,b,c\}))$$
  1. posso generalizzare?

In generale,

$\forall |S| > 2,\ max\_set(S)=\max(s \in S, max\_set(S\setminus \{s\})$

Posso quindi definire la seguente funzione ricorsiva:

$$ max\_set{(S=\{s_1, s_2, \dots, s_n\})} = \begin{cases} s, & \text{se } S=\{s\} \text{ (i.e. |S|=1)}\\ \max(s_1,s_2), & \text{se } S=\{s_1,s_2\}\\ \max(s\in S, max\_set(S\setminus \{s\})), & \text{se } |S|>2 \end{cases} $$

Che pu essere ulteriormente semplificata come:

$$ max\_set{(S=\{s_1, s_2, \dots, s_n\})} = \begin{cases} s, & \text{se } S=\{s\} \text{ (i.e. |S|=1)}\\ \max(s\in S, max\_set(S\setminus \{s\})), & \text{se } |S|\geq 2 \end{cases} $$

Ricerca del massimo in un insieme: un altro approccio ricorsivo¶

Posso pensare di dividere la ricerca del massimo in un insieme come la ricerca ricorsiva dei massimi locali su due met dell'insieme, e prendere quindi il valore pi grande tra i due.

In altri termini:

  • partiziono l'insieme in due met
  • calcolo il massimo $m_1$ nella prima met
  • calcolo il massimo $m_2$ nella seconda met
  • calcolo il massimo tra $m_1$ ed $m_2$

Per comodit, vedo l'insieme come un array $\mathbf{v}$ ed il numero di elementi che lo compone com $\#\mathbf{v}$.

Posso quindi definire la funzione massimo in questo modo:

$$ max\_set(\mathbf{v}) = \begin{cases} v_1 & \text{se }\#\mathbf{v} = 1,\\ \max( max\_set(\mathbf{v}_{1,\dots,\lfloor{\frac{\#\mathbf{v}}{2}}\rfloor}),\ max\_set(\mathbf{v}_{\lfloor{\frac{\#\mathbf{v}}{2}}\rfloor+1, \dots, \#\mathbf{v}})) & \text{se }\#\mathbf{v} > 1 \end{cases} $$

In questo modo, ho due invocazioni ricorsive nella stessa funzione.

In [32]:
#include <stdio.h>
int max_set(int v[], int n) 
{
    if(n == 1)
        return v[0];
    
    // cerco il max sulla prima met di v
    int m1 = max_set(v, n/2);
    // cerco il max sulla seconda met di v (pi eventuale elemento rimanente)
    int m2 = max_set(v+n/2, n/2+n%2);
    if (m1 >= m2)
        return m1;
    return m2;
}
            
int main()
{
  int v[] = {7,6,8,3,9,11,2,4,13,21,24,1,3};
  int m   = max_set(v, 13);
  printf("il massimo  %d\n",m);
  return 0;
}

In generale, dato un problema $P$, un insieme di dati $D_t$ con $t$ "indice di complessit" dell'insieme di dati (i.e., $t+1 > t$ $\Rightarrow$ problema su $D_{t+1}$ pi complesso che su $D_{t}$), una soluzione $S$ del problema $P$ ricorsiva se pu essere definita come: $$S(D_t)=\begin{cases} S_0 & \textit{se } D_t=D_0,\\ f(S(D_{j}), D_t) & \textit{se } D_t\neq D_0 \land j < t \end{cases}$$ Dove $D_0$ un insieme di dati su cui il problema ha una soluzione $S_0$ banale/nota ed $f$ una funzione in grado di sfruttare una soluzione su un insieme di dati pi semplice e, se necessario, i dati con complessit $D_t$.

Problema: ricerca di un elemento $k$ in un insieme (e.g. $S=\{7,6,2,8,4\}, k=8$). Si desidera una funzione che restituisca un valore logico di

  • true (e.g., $1$), se $k \in S$,
  • false (e.g., $0$) altrimenti.

Soluzione iterativa: scorro l'insieme finch non trovo l'elemento selezionato. Se non lo trovo, l'elemento non presente. Restituisco $1$ se presente, $0$ altrimenti.

(Un') implementazione iterativa:

int S[]     = { 7,6,2,8,4 };
int n       = 5;
int k       = 8;
int trovato = 0;
for(int i = 0; i < n; i=i+1)
{
    if (S[i] == k)
    {
        trovato = 1;
        break; 
    }
}

Soluzione ricorsiva:

Voglio definire una funzione $find\_in\_set(S, k)$, con $S$ insieme e $k$ valore da cercare.

  1. esistono dei casi in cui il mio problema facilmente risolvibile?
    • Se il mio insieme vuoto, ovviamente l'elemento non presente $\Rightarrow \forall k,\ find\_in\_set(\emptyset, k) = false$
    • Se l'insieme composto da un solo elemento, allora: $$find\_in\_set(S=\{s\}, k) = \begin{cases} false & \text{ se } s \neq k\\ true & \text{ altrimenti}. \end{cases}$$
  1. posso sfruttare le soluzioni ai casi banali individuati per costruire soluzioni ad un caso un po' pi complesso?

Se $S=\{s_1,s_2\}$, posso risolvere il problema sui due sottoinsiemi $\{s_1\}$ ed $\{s_2\}$ che, avendo cardinalit $1$, so risolvere. Se almeno uno dei due mi ha dato esito $true$, allora l'elemento $k$ presente in $S$, altrimenti no.

$$find\_in\_set(S=\{s_1,s_2\}, k) = find\_in\_set(\{s_1\}, k) \lor find\_in\_set(\{s_2\}, k)$$
  1. posso sfruttare la mia nuova soluzione per costruire una soluzione ad un caso ancora pi complesso?

Posso sfruttare una soluzione di $|S|\leq 2$ per $|S| = 3$? Ripetendo il ragionamento precedente, posso dividere $S$ in due sottoinsiemi: $$\{s_1\} \text{ e } \{s_2, s_3\}$$ per ognuno dei quali so trovare la soluzione. Anche in questo caso, la soluzione del problema con $|S|=3$ sar data dall'$or$ logico tra le soluzioni dei due sottoinsiemi. $$find\_in\_set(S=\{s_1,s_2, s_3\}, k) = find\_in\_set(\{s_1\}, k) \lor find\_in\_set(\{s_2,s_3\}, k)$$

  1. posso generalizzare?
$$\forall |S|>1, find\_in\_set(S, k) = find\_in\_set(\{s \in S\}, k) \lor find\_in\_set(S \setminus \{s\}, k)$$

Alcuni tipi di ricorsione¶

In base al numero di invocazioni ricorsive che si trovano all'interno di una funzione, si distingue tra:

  • ricorsione lineare: sufficiente una singola chiamata ricorsiva per definire il valore restituito
return find_in_set(S+1, k, n-1);
  • ricorsione non lineare: sono necessarie pi chiamate ricorsive per definire il valore restituito
    return find_in_set(&S[0], k, 1) || find_in_set(S+1, k, n-1);

Ordinamento per selezione¶

dato un vettore $\mathbf{v}$ di reali, ordinare il vettore implementando una soluzione ricorsiva che sfrutti l'ordinamento per selezione (Una) versione iterativa:

float idx_min_array(float v[], int n, int start)
{
    int idx_min = start;
    float min   = v[start];
    for (int i=start+1; i<n; i++)
        if (v[i] < min)
        {
            min     = v[i];
            idx_min = i;
        }
    return idx_min;
}

void selection_sort(float v[], int n)
{
    int idx_min;
    for(int i=0; i<n-1; i++) // n-1, in quanto ultimo elemento,
                             // sar gi nella posizione corretta
                             // grazie agli spostamenti precedenti
    {
        idx_min    = idx_min_array(v, n, i);
        float tmp  = v[i];
        v[i]       = v[idx_min];
        v[idx_min] = tmp;
    }

}

Voglio provare a trovare una soluzione ricorsiva.

  1. esistono dei casi in cui il mio problema facilmente risolvibile?
  • Se $\mathbf{v}$ ha un solo elemento, la versione ordinata del vettore banalmente $\mathbf{v}$ stesso. Quindi,
    • se $\#\mathbf{v} = 1$, $selection\_sort(\mathbf{v}) = \mathbf{v}$

  • Se $\mathbf{v}$ ha due elementi, i.e.$\mathbf{v}=(v_1,v_2)$, la versione ordinata di $\mathbf{v}$ sar:

    • se $\min(v_1, v_2) = v_1$, $\mathbf{v}^{sorted}=(v_1,v_2)$

    • se $\min(v_1, v_2) = v_2$, $\mathbf{v}^{sorted}=(v_2,v_1)$.

      Posso quindi definire la funzione selection_sort per un vettore di due elementi come:

    • $selection\_sort(\mathbf{v}) = (v_{idx\_min}, v_j)$ con $v_{idx\_min} \leq v_j$.
  1. posso sfruttare le soluzioni ai casi banali individuati per costruire soluzioni ad un caso un po' pi complesso?
  • analizziamo il caso in cui $\mathbf{v}=(v_1,v_2, v_3)$.

In questo caso, se si vuole sfruttare il principio di base del Selection Sort (ossia la ricerca del minimo), la versione ordinata del vettore la posso esprimere come il vettore composto dalla concatenazione di:

  • $v_{idx\_min}$, con $idx\_min$ indice del valore minimo in $\mathbf{v}$
  • l'ordinamento del sottoarray formato dalle 2 componenti rimanenti, che so calcolarlo.

Indicando con $\mathbf{v}^{i \neq t}$ il vettore composto da tutte le componenti di $\mathbf{v}$ tranne la $t$-esima (ossia $\mathbf{v}^{i \neq t}=(v_1,v_2,\dots,v_{t-1},v_{t+1},\dots,v_{\#\mathbf{v}})$) posso riscrivere la funzione $selection\_sort(\mathbf{v})$ su array di dimensione 3 come:

$$selection\_sort(\mathbf{v}) = concat\big(v_{idx\_min}, selection\_sort(\ \mathbf{v}^{i \neq idx\_min}\ )\big)$$

con $concat(\dots)$ funzione che restituisce la concatenazione dei due vettori dati in input.

  1. posso sfruttare la mia nuova soluzione per costruire una soluzione ad un caso ancora pi complesso?

// soluzione ricorsiva
#include <stdio.h>
#define MAX_LEN 15

int idx_min(int v[], int n)
{
    int idx = 0;
    for(int i=1; i < n; i++)
        if (v[i] < v[idx])
            idx = i;
    return idx;
}

void swap(int* a, int* b)
{
    int t = *a;
    *a    = *b;
    *b    = t;
}

void selection_sort(int v[], int n)
{
   if(n==0 || n==1)
       return;
    
    int idx = idx_min(v,n);

    swap(&v[0], &v[idx]);
    selection_sort(v+1, n-1);
}
